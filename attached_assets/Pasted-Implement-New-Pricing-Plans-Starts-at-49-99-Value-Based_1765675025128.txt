6) Flagged Review Trigger Logic (must be deterministic)
Flow on resume upload / analysis:

Run baseline analysis (cheap model(s))

Compute risk_score (0–100) from:

timeline overlaps/gaps

name/email mismatch

skill soup / unrealistic breadth

too-perfect indicators

role/tenure mismatch

If risk_score >= FLAG_THRESHOLD THEN attempt advanced review:

Check per-candidate flagged cap

Check org monthly flagged cap

Check auto-flag rate cap (flagged_count <= floor(candidate_count * cap_pct))

If allowed, run flagged review using gpt-4.1

If not allowed, mark:

advanced_review_status = "skipped_quota"

Show UI note
