1. Database changes

Create/update an onboarding_plans table with:

id

candidate_id (nullable)

employee_name

role

start_date

onboarding_type

status ("active" or "completed")

plan_json (full AI output, including checklist, 30/60/90, emails)

completed_task_ids (JSON array of checklist task IDs, default empty)

created_at

completed_at (nullable)

When a plan is generated successfully, insert a row with:

status="active"

plan_json = AI response

completed_task_ids = []

2. Checklist task IDs

When generating the checklist in plan_json, ensure each task has a stable id string (e.g. "week1_task1", "week1_task2", etc.).

The frontend will use these IDs to mark tasks done.

3. APIs

GET /api/onboarding/plans?status=active
Returns all active plans with basic fields plus computed task counts if convenient.

POST /api/onboarding/plans/:id/tasks with body { "task_id": "week1_task1", "completed": true|false }

If completed=true, add task_id to completed_task_ids (if not already there).

If completed=false, remove it from completed_task_ids.

POST /api/onboarding/plans/:id/complete
Sets status="completed" and completed_at=now().

4. Frontend – Active Plans list

On the /onboarding page, above the existing form, add a section “Active Onboarding Plans” with a table:

Columns: Employee, Role, Start Date, Progress, Actions.

Populate via GET /api/onboarding/plans?status=active.

Progress shows: <completed_tasks_count> / <total_tasks_in_checklist>.

Actions:

View:

Loads that plan into the existing form:

Fill employee name, role, start date, candidate if present, onboarding type.

Parses plan_json and renders Checklist / 30/60/90 / Emails as we already do for a freshly generated plan.

Uses completed_task_ids to mark checklist tasks visually as done.

Mark Completed:

Calls POST /api/onboarding/plans/:id/complete.

On success, remove the row from the table.

5. Frontend – Checklist task toggles

In the Checklist tab, add a small checkbox or “Mark as done” control on each task card.

Toggling this calls POST /api/onboarding/plans/:id/tasks with the task’s id and the new completed value.

Style completed tasks (e.g. check icon + lighter background) and update the progress in the Active Plans list.

6. Constraints

Do not change the current layout of the Onboarding form or tabs.

Do not change the way we generate the AI plan beyond adding task IDs.

Keep everything visually consistent with the existing design.

That gives you:

All candidates with onboarding in one simple list.

Minimal state: plan active/completed + which tasks are done.

No crazy workflow engine.
